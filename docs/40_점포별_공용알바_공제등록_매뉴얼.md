# 점포별 공용알바 공제등록 — 사용 매뉴얼

> 화면 ID: `신규개발화면` · 작성팀: 상품지원팀 · 화면 메뉴: **점포별 공용알바 공제등록**
> 사이드바 → `점포별 공용알바 공제등록` 으로 진입

이 화면에서는 **점포별 인건비를 매입처(협력사)들에게 매출 점유율 기준으로 공제·배분**하는 작업을 수행합니다. 등록 후 페이지 41(공제확정)에서 최종확정·IFAS 전송으로 이어집니다.

---

## 0. 한눈에 보는 작업 흐름

```
[1] 조회 ─→ [2] 공제정보 신규/수정 ─→ [3] 공제정보 그리드에서 선택
                                                   │
[4] 매입처 입력바 (코드/명/유형/점유율/공제액)         │
       ▼                                            │
[5] 매입처 그리드 ◀───── 매출조회 / 파일등록 / 일괄적용
       │
       ▼
   확정(상태=확정) ─→ 페이지 41에서 최종확정·IFAS 전송
```

---

## 1. 조회 영역 (상단 필터)

| 항목 | 형식 | 기본값 | 비고 |
|---|---|---|---|
| **정산년월** ＊ | 월선택 | 당월 | 필수, `YYYY-MM` |
| 영업점 | 풀다운 | 전체 | 영업점 리스트에서 선택 |
| **진행상태** ＊ | 풀다운 | 전체 | 전체 / 작성중 / 확정 |

### 버튼
- **조회**: 입력 조건으로 공제정보 그리드 필터링. 첫 행을 자동 선택해 폼을 채움.
- **초기화**: 조회 조건을 기본값으로 되돌림.

---

## 2. 공제정보 입력 폼

> 신규 마스터(공용알바공제번호) 채번 또는 기존 건 수정에 사용.

| 항목 | 형식 | 비고 |
|---|---|---|
| **정산년월** ＊ | 월선택 | 확정 상태에선 수정 불가 |
| **영업점** ＊ | 풀다운 | 확정 상태에선 수정 불가 |
| **총인건비** ＊ | 숫자 | **공급가 기준으로 입력**. 천단위 콤마 자동. VAT 10% 자동 가산 후 그리드/계산에 반영 |

### 버튼

| 버튼 | 동작 |
|---|---|
| **초기화** | 폼 비움 + 선택 해제 (그리드 데이터엔 영향 없음) |
| **저장** | 선택 행이 없거나 확정 상태일 때 → **신규 채번**. 작성중 상태의 선택 행이 있으면 → **기존 건 업데이트** |
| **재생성** | 선택 행의 매입처 설정(유형/점유율/공제액)은 유지하고 매출·매출제외만 0으로 초기화. 그 후 [매출조회] 또는 [파일등록] 필요 |
| **삭제** | 그리드에서 체크된 마스터 일괄 삭제. 확정 건 포함 시 거부 |
| **확정** | 선택 행 상태를 `작성중`→`확정`. 확정 후엔 매입처 정보 수정·삭제 불가 |

### 채번 규칙
형식: `AB` + `YYYYMM` + `점포코드(3)` + `일련번호(5)` = **총 16자리**
> 예시: 광화문점(001), 2026-03 첫 건 → `AB20260300100001`

---

## 3. 공제정보 그리드

| 컬럼 | 설명 |
|---|---|
| 공용알바공제번호 | 미채번 시 공란, 채번 후엔 자동 표시 |
| 정산년월 / 영업점 | 입력값 |
| **총인건비** | 공급가 (입력값 그대로) |
| **총인건비(VAT포함)** | `총인건비 × 1.1`. 실제 계산 기준값 |
| 진행상태 | 작성중(파랑) / 확정(빨강) |
| 정산등록일·사번·자명 | 신규 채번 시 자동 기록 |
| 비고 | **인라인 입력 가능** (확정 건은 수정 불가) |

> 행을 클릭하면 폼이 채워지고 매입처 그리드도 해당 마스터의 매입처들로 갱신됩니다.

---

## 4. 공제매입처 입력바

매입처를 그리드에 추가하거나 일괄적용을 위한 영역.

| 항목 | 동작 |
|---|---|
| **매입처코드** | 입력 후 Enter → 해당 점포의 매입처 풀에서 정확 일치 → 즉시 추가. 불일치 → 매입처 조회 모달 |
| **매입처명** | 입력 후 Enter → 부분 일치. 1건이면 즉시 추가, 다수/없음이면 모달 |
| 🔍 (조회 버튼) | 매입처 조회 모달 직접 열기 |
| ☑ 공제유형 + 풀다운 | `일반/고정점유율/고정공제/제외` 중 선택 |
| ☑ 점유율 + 입력 | 고정점유율(R)일 때 사용. 정수 3자리 + 소수점 2자리까지 (예: `8.5`, `12.34`, `999.99`) |
| ☑ 공제액 + 입력 | 고정공제(A)일 때 사용. 천단위 콤마 |

### 버튼

| 버튼 | 동작 |
|---|---|
| **일괄적용** | 그리드에서 체크된 매입처 다건에 대해, 위 체크박스로 선택한 항목들을 동시 변경 |
| **매출조회** | 정산년월 기준 전월 매출을 DW에서 일괄 조회하여 매출금액 컬럼에 반영. 기존 매출제외금액은 유지 |
| **파일등록** | 엑셀(xlsx/xls) 업로드로 매입처 일괄 등록 (자세한 양식은 §6 참조) |
| 초기화 | 입력바 필드만 초기화 |
| 엑셀다운 | 현재 그리드 내용을 엑셀로 다운로드 |
| 저장 | 매입처 변경사항 저장 (UI 알림용) |
| 삭제 | 체크된 매입처 행 삭제 |

### 매입처 추가 시 동작 (중요)
- 코드/명만 자동 채워짐, **매입처품목코드명은 기본값 `일반`**
- **매출/매출제외/공제액은 0** (사용자가 매출조회 또는 파일등록 클릭 후에만 채워짐)
- 매출이 채워지면 점유율·공제액이 자동 산정됨

### 중복 등록 규칙 / 매입처품목 자동 채번
- 중복 체크는 **`(매입처코드, 매입처품목코드)` 조합** 단위
- 동일 매입처를 **다른 매입처품목코드**로는 추가 가능 (예: BST 일반 + BST 기획전)
- 매입처코드 추가 시 카탈로그의 **첫 품목(`일반`)부터 순서대로** 사용 가능한 첫 품목을 자동 선택
  - 이미 일반이 등록되어 있으면 → 다음 품목(예: `대표상품`) 자동 사용
  - 모든 매입처품목이 이미 등록되어 있으면 알림 후 추가 차단
- 기존 행의 매입처품목 dropdown으로 변경 시에도 충돌 검사: 같은 매입처의 다른 행과 매입처품목이 겹치면 변경 차단

## 5. 공제매입처 그리드 (계산 결과)

| 컬럼 | 편집 가능 여부 / 산식 |
|---|---|
| ☑ 체크 | 다중 선택 — 일괄적용/삭제 대상 |
| 순번 | 행 순서 자동 |
| 매입처코드 / 매입처명 | 마스터에서 자동 |
| 매입처품목코드 | 매입처품목코드명 dropdown 선택과 연동 |
| **매입처품목코드명** | dropdown — 해당 매입처에 매핑된 여러 품목 중 선택. 기본값 `일반` |
| **유형** | dropdown — 일반(N) / 고정점유율(R) / 고정공제(A) / 제외(E) |
| 매출금액 | DW 조회/파일 업로드로 채워짐 |
| **매출제외금액** | **인라인 입력 가능** — 매출 중 제외할 행사금액 등 |
| 최종매출금액 | `매출금액 − 매출제외금액` |
| **점유율(%)** | R: **인라인 입력**. N: 자동 (`자사매출/일반총매출×100`). A: `-` |
| **공제액** | A: **인라인 입력**. N/R: 자동 산정 |
| 진행상태 | 마스터 진행상태 표시 |
| 등록일·사번·자명 | 매입처 추가 시점 |
| 수정일·사번·자명 | 매입처 변경(일괄적용/인라인) 시점 |

### 공제액 산출식

| 유형 | 산식 |
|---|---|
| 고정공제 (A) | `공제액 = 입력 공제액` (매출·인건비 무관) |
| 고정점유율 (R) | `공제액 = 자사 최종매출 × 고정점유율` |
| 일반 (N) | `공제액 = 자사 최종매출 × (분배목 ÷ 일반형 총 최종매출)` |
| 제외 (E) | 공제대상 제외 (`-`) |

**일반형 분배목 = `총인건비(VAT포함) − 고정공제(A) 합계 − 고정점유율(R) 공제 합계`**

> 모든 공제액은 1원 단위로 반올림 (`Math.round`).
> R/A가 1건 이상 있으면 그리드 합계 행 위에 노란색 **공제 산출 근거 행**이 자동으로 노출되어 산식이 어떻게 적용됐는지 한눈에 검증할 수 있습니다.

### 계산 예 (가상 시나리오)
- 총인건비 공급가: 3,590,689원 → 총인건비(VAT포함): **3,949,758원**
- A협력사 (R 8%, 매출 3,692,540): 3,692,540 × 0.08 = **295,403원**
- B협력사 (R 5%, 매출 3,933,560): 3,933,560 × 0.05 = **196,678원**
- R 합계 = 492,081원 → 분배목 = 3,949,758 − 492,081 = **3,457,677원**
- 일반(N) 매입처들이 위 분배목을 매출 비중대로 나눠 가짐

## 6. 파일등록 (엑셀 업로드) 양식

업로드 가능한 컬럼 (헤더명을 자동 감지, 위치 무관):

| 헤더명 (포함 인식) | 필수 | 비고 |
|---|---|---|
| `매입처코드` 또는 `거래처코드` 또는 `코드` | ✅ | 영숫자 4~10자 (예: `0803176`, `01B0470`) |
| `매입처명` 또는 `거래처명` |  | 마스터에 없는 코드면 표시 이름으로 사용 |
| `매출` 또는 `매출금액` |  | `2,223,710` 같이 콤마/공백 포함 OK |
| `매출제외금액` 또는 `매출제외` |  | 콤마/공백 포함 OK |
| `매입처품목코드` |  | 카탈로그에 매칭되면 그 품목으로, 아니면 `일반` |
| `매입처품목코드명` |  | 정보용 (보조) |

### 동작 규칙
- 헤더에 `매입처코드`가 없으면 거부 (알림)
- 이미 그리드에 있는 코드는 건너뜀 (알림에 중복 N건으로 표기)
- 영숫자 코드 허용 (`01B0470` 등 음반 코드도 OK)
- 매출 셀의 통화 기호/콤마는 자동 제거 후 숫자로 변환

### 예시 양식
```
매입처코드  매입처명          매출
0803176     라이브워크        2,223,710
0817560     도나앤데코        1,429,400
0806606     디자인랩(레더랩)  877,500
0803229     웨이크업(wakup)   227,600
0803288     명성코리아        166,950
```

## 7. 자주 쓰는 시나리오

### A. 신규 공제 작성 (예: 창원점 2026-03)
1. 폼: 정산년월 `2026-03`, 영업점 `창원점`, 총인건비 `3,590,689` 입력 → **저장**
2. 신규 공용알바공제번호 자동 채번 → 그리드에 행 생성
3. 매입처 입력바로 매입처 추가 또는 **파일등록**으로 일괄 추가
4. **매출조회** 클릭 → DW 매출 일괄 반영 → 자동 점유율·공제액 산정
5. 특정 매입처를 R/A로 변경하려면 그리드의 유형 dropdown 변경 + 점유율/공제액 인라인 입력
6. 검증 후 **확정** → 페이지 41(공제확정)로 이동해 최종확정·IFAS 전송

### B. 매출 반영 잘못된 경우
- **재생성** 클릭 → 매출 0으로 초기화 (매입처 설정은 유지)
- **매출조회** 또는 **파일등록**으로 다시 채움

### C. 매출제외(행사금) 반영
- 그리드 매입처 행의 `매출제외금액` 셀에 직접 입력
- 최종매출/점유율/공제액이 자동 갱신됨

### D. 일부 매입처에 고정 정책 적용 (일괄)
1. 그리드에서 대상 매입처들 ☑ 체크
2. 입력바: ☑공제유형 = `고정점유율`, ☑점유율 = `5.5` 입력
3. **일괄적용** 클릭 → 체크된 행이 동시에 변경됨

---

## 8. 권한·검증

- **확정 건**은 모든 입력 필드 비활성화 (수정·삭제 불가)
- **재생성**: 확정 건은 거부
- **삭제(마스터)**: 확정 건이 1건이라도 포함되면 거부
- 페이지 41에서 IFAS 전송된 건은 페이지 40에서도 변경 불가 상태로 보임

---

## 9. 페이지 41과의 동기화

페이지 40에서 신규 생성/수정/확정한 건은 즉시 페이지 41 그리드에도 반영됩니다 (전역 컨텍스트 공유).
- 페이지 40 → 41: 신규/수정/확정 즉시 반영
- 페이지 41 → 40: 최종확정/확정취소/IFAS전송 결과가 페이지 40에서 거꾸로 보임

## 10. 참고 데이터 파일

- `점포별_공용알바_공제매입처_리스트.xlsx`: 매입처마스터 + 점포별 매출 매트릭스 + 점포별 시트(광화문/강남/잠실/부산/창원). 매입처별 다중 품목(일반 + 추가 카테고리) 구조. 파일등록의 양식 참고용으로 활용 가능.
